# Sigil Grid (руны) на WebGL

Вариант игры в духе **2048**: на поле отображаются **руны-сигилы** (эмодзи) вместо чисел, но внутренняя математика слияний та же — две одинаковые плитки сливаются в следующий уровень. Реализация на чистом JavaScript без фреймворков:

- рендер плиток и символов выполняется через WebGL (символы рисуются как текстуры с 2D-canvas);
- интерфейс победы/поражения и **легенда порядка слияний** — через HTML/CSS;
- управление — стрелками клавиатуры.

## Состав проекта

В проекте 3 основных файла:

- `index.html` — структура страницы, блок `#gameStage` с полем и боковой панелью «Merge order», UI-слои победы и поражения.
- `styles.css` — визуальное оформление страницы, кнопок, оверлеев и таблицы легенды.
- `game.js` — игровая логика, сопоставление значений с рунами (`RUNE_BY_VALUE`), WebGL-рендер и обработчики событий.

## Как запустить

### Вариант 1 (самый простой)

1. Откройте `index.html` в браузере двойным кликом.
2. Игра сразу готова: используйте стрелки `← ↑ → ↓`.

### Вариант 2 (через локальный сервер, рекомендовано)

Некоторые браузеры стабильнее работают с WebGL при запуске через localhost.

В папке проекта выполните:

```bash
python -m http.server 8000
```

После этого откройте в браузере:

`http://localhost:8000`

## Управление

- `ArrowLeft` / `ArrowRight` / `ArrowUp` / `ArrowDown` — движение плиток.
- `Начать заново` — перезапуск партии.
- На экране победы (достигнут финальный сигил 🔮, значение 2048):
  - `Продолжить` — продолжить после победы;
  - `Новая игра` — начать заново.
- На экране поражения:
  - `Попробовать снова` — начать новую игру.

Справа от поля таблица **Порядок величин** показывает цепочку слияний от первой руны до финальной.

## Подробный разбор файлов

## `index.html`

Файл задаёт каркас интерфейса:

- Блок `#ui`:
  - заголовок **Игра слияния**;
  - текущий счёт (`#score`).
- Блок `#gameStage`:
  - слева/центр — `#board` с `<canvas id="glcanvas">`, оверлеями `#winLayer` и `#gameOverLayer`;
  - справа — `<aside class="rune-legend">` с таблицей шагов и сигилов (`#runeLegendBody` заполняется из JS).
- Кнопка `#restart` под сценой.
- В конце подключается `game.js`.

Идея: HTML хранит оболочку и легенду; игровая математика и отрисовка — в `game.js`.

## `styles.css`

Файл отвечает за внешний вид:

- Компоновка `#gameStage`, поля и боковой панели с легендой.
- Размер поля по-прежнему фиксирован (`420×420` у canvas в логике игры).
- Оверлеи `#winLayer` и `#gameOverLayer` поверх поля, переключаются классом `.hidden`.
- Стили таблицы рун и прокрутки легенды.

## `game.js`

Ниже — основные части по смысловым блокам.

### 1) Руны и константы

- Объект `RUNE_BY_VALUE` сопоставляет внутренние степени двойки (`2`, `4`, `8`, …, `2048`) с отображаемым эмодзи на плитке.
- `WIN_VALUE = 2048` — порог победы (финальный сигил 🔮).
- `runeLabel(value)` — что именно рисовать на плитке; для неизвестных значений — число как строка.
- `populateRuneLegend()` — заполняет таблицу порядка слияний при загрузке.

Дальше: `SIZE = 4`, `CANVAS_SIZE = 420`, `GAP = 10`, ссылки на DOM (canvas, счёт, кнопки, оверлеи).

### 2) Инициализация WebGL

- `const gl = canvas.getContext("webgl")`;
- два набора шейдеров: **цвет** для прямоугольников плиток и **текстура** для отрисовки символа на плитке;
- `createShader` / `createProgram`, буферы `positionBuffer`, `texcoordBuffer`.

### 3) Состояние игры

- `grid` — матрица `4×4` с **внутренними** значениями плиток (как в классическом 2048);
- `score`, `gameOver`, `winShown`, `winOverlayVisible` — как раньше.

### 4) Новые плитки

- `addRandomTile()` — с вероятностью 1% ставит 💣, с вероятностью 5% - 🟡, с вероятностью 84% ставит `2`, иначе `4` (на экране — соответствующие первые руны из таблицы).

### 5) Ходы и слияние

Логика как в 2048: `compressAndMerge`, `moveLeft`, повороты сетки для остальных направлений, `move(direction)` — после изменения поля добавляется новая плитка, проверяются победа (`hasWinningTile` по `WIN_VALUE`) и проигрыш (`isGameOver`).
Также добавлены 2 новые плитки: 💣 бомба уничтожает любую плитку при слиянии, 🟡 золотая плитка улучшает любую другую плитку при слиянии.

### 6) Отрисовка

- **Плитки** — `drawRect` + `getColor(value)` по внутреннему значению.
- **Символ на плитке** — `getTextTexture`: текст (в т.ч. эмодзи) рисуется на скрытом 2D-canvas с шрифтом `EMOJI_FONT_STACK`, загружается в WebGL-текстуру, кэш `textTextureCache`;
  `fontSizeForTileGlyph` подбирает размер для руны или длинного числа.

### 7) `draw()`

Сначала цветные плитки без blending, затем с blending — текстуры символов поверх.

### 8) Сброс и события

- `resetGame()` — новая партия, две стартовые плитки, первый `draw()`.
- Стрелки, кнопки перезапуска и продолжения после победы — как в классической версии.

В конце вызываются `populateRuneLegend()` и `resetGame()`.
